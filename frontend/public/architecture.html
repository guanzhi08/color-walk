系統架構、服務流程與技術選型一覽

🏗️ 系統架構

前端 Frontend

React 18 + Vite 5
Tailwind CSS 3
React Router v6
React Leaflet 地圖
Axios HTTP 客戶端
SVG 顏色輪盤

後端 Backend

FastAPI + Uvicorn
SQLAlchemy 2.0 ORM
Pydantic v2 資料驗證
純 Python JWT (HS256)
bcrypt 密碼雜湊
Pillow 影像處理

外部服務

Neon.tech PostgreSQL
Cloudinary 圖片儲存
OpenStreetMap 地圖
Render.com 部署
UptimeRobot 喚醒
單一服務架構： 前端 React 應用程式以 Vite 打包後輸出至 backend/static/， 由 FastAPI 的靜態檔案路由直接提供，所有 /api/* 請求打到後端， 其餘路徑回傳 index.html 做 SPA fallback。 前後端同源，無需 CORS 設定。

🔧 技術選型說明
功能選擇原因
JWT 驗證 hmac + hashlib python-jose 與系統 cryptography 衝突；pure Python HS256 無依賴
密碼雜湊 bcrypt 直接使用 passlib 在新版 bcrypt 有 __about__ AttributeError
資料庫驅動 psycopg[binary] 3.3.4 psycopg2-binary 在 Python 3.14 有 undefined symbol；psycopg3 原生支援
EXIF 擷取 getexif().get_ifd(0x8825) 現代 Pillow API；舊 _getexif() 已棄用。Samsung 手機 NaN 座標需過濾
GPS Fallback 瀏覽器 Geolocation API Samsung 等 Android 手機拍照 EXIF 可能為 NaN；改用瀏覽器定位
前後端整合 Vite build → FastAPI static 單一 Render 服務，同源避免 CORS，免費方案更容易管理
地圖 React Leaflet + OSM 免費、無 API Key 限制、輕量
管理員建立 環境變數自動建立 Render 免費方案無 Shell；首次啟動若 DB 為空則自動建立管理員帳號
🚀 部署流程
  • 1

    Neon.tech 建立資料庫
    免費 PostgreSQL，複製連線字串（含 ?sslmode=require）備用。

  • 2

    Cloudinary 取得憑證
    Cloud Name、API Key、API Secret 填入 Render 環境變數。

  • 3

    GitHub Push 觸發 Render 部署
    Render 執行 Build Command：cd frontend && npm run build → pip install -r requirements.txt。

  • 4

    自動建立管理員帳號
    設定 ADMIN_USERNAME + ADMIN_PASSWORD 環境變數，首次啟動 DB 為空時自動新增管理員。

  • 5

    UptimeRobot 設定監控
    監控 https://color-walk.onrender.com/health，每 5 分鐘 ping 一次防止 Free Tier 休眠。
